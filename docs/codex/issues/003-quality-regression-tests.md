# Issue 003: 品質回帰テストと NG サンプル検証を追加する

## Background

現状は生成品質をコードで守っていないため、語彙やテンプレート更新で違和感のあるお題が戻りやすい。

## Scope

- ワード抽出の検証コード
- NG サンプル定義
- 必要なら `package.json`

## Tasks

- seed 固定または再現可能な形で生成を検証できる仕組みを作る
- NG サンプルと期待サンプルを fixture 化する
- 代表的な NG 例が再生成されないことを確認できるようにする
- ローカルで回しやすい検証コマンドを追加する

## Acceptance Criteria

- 代表的な NG サンプルを検知できる
- 語彙更新時にローカルで品質確認できる
- 実装を壊しにくい最小限の自動検証がある

## Example NG Cases

- `コンビニ店員なのにスマホだけ異常に使いこなしているおばあちゃん。なぜそんなことに？`
- `理容師なのにフードデリバリーアプリだけ異常に使いこなしている転校生。なぜそんなことに？`
- `飼育員が教室でだけ見せる裏の顔`
- `温泉旅館でレビューを書いている転校生を見た周囲の反応`
